Uploading files to ORArchive is the fastest way to get your documents, images, and media into your archive. You can upload files by dragging them directly onto the interface, using the built-in file picker, or queuing an entire batch at once — each uploaded file is automatically converted into a searchable record that you can enrich with metadata right after the upload completes.

## Uploading a Single File

<Steps>
  <Step title="Open the upload dialog">
    Click the **Upload** button in the top navigation bar, or drag any file from your desktop directly onto the main content area. Either action opens the upload dialog.
  </Step>
  <Step title="Select your file">
    In the upload dialog, click **Choose File** to open your device's file picker and select the file you want to upload, or drag the file directly into the dialog's drop zone. The file name and size are shown in the dialog once selected. Click **Start Upload** to begin.
  </Step>
  <Step title="Monitor the upload progress">
    Your file appears in the upload dialog with a progress bar showing the percentage completed. You can continue navigating ORArchive while the upload runs in the background — the dialog minimizes to a status indicator in the bottom-right corner.
  </Step>
  <Step title="Add metadata to your new record">
    Once the upload is complete, ORArchive automatically opens the record detail panel for your new file. From here, you can add a title, description, category, and tags to make the record easier to find later.
  </Step>
</Steps>

## Uploading Multiple Files at Once

When you have a large batch of files to archive, you can upload them all in a single operation. In the file picker, hold **Ctrl** (Windows/Linux) or **Cmd** (macOS) and click each file you want to select, then click **Open**. Alternatively, select a range of files by clicking the first file, then holding **Shift** and clicking the last file in the range.

You can also drag an entire folder or group of selected files from your desktop and drop them onto the content area at once. ORArchive creates a **separate record for each file** — files are never merged into a single record during a batch upload.

All queued files appear in the **Batch Upload Queue** panel, where each file has its own progress bar and status indicator. Files upload concurrently up to a system-determined limit, and completed items are marked with a green checkmark. If any file fails — for example, because its format is unsupported — it is flagged in red with an explanation, and the rest of the queue continues uninterrupted.

Once every file in the batch has finished, you can click **Review All Records** in the queue panel to open a bulk metadata editor where you can assign a shared category or tags to the entire batch at once, saving you time on repetitive data entry.

## Supported File Types

ORArchive accepts files across a wide range of categories. Refer to the table below to confirm your format is supported before uploading.

| Category | Formats |
|---|---|
| Documents | PDF, DOCX, DOC, XLSX, XLS, PPTX, TXT, RTF |
| Images | PNG, JPG, JPEG, GIF, TIFF, SVG, WEBP |
| Video | MP4, MOV, AVI, MKV |
| Audio | MP3, WAV, AAC, FLAC |
| Archives | ZIP, RAR, 7Z |

If you need to archive a file type not listed here, contact ORArchive support. Unsupported file types will be rejected at upload time with an error message indicating the reason.

<Note>
  Each individual file must be **2 GB or smaller**. Your workspace includes **10 GB of free storage**. You can monitor your current usage at any time under **Settings → Storage**. Paid plans offer expanded storage limits and higher per-file size caps.
</Note>

<Tip>
  Enable browser notifications for ORArchive so you receive an alert the moment a large upload finishes — even if you've switched to another tab or application. You can turn this on the first time you start an upload, or anytime under **Settings → Notifications → Browser Alerts**.
</Tip>
