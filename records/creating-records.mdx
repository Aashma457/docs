In ORArchive, a **record** is the fundamental unit of archived content. A record can be an uploaded file, a text note you compose directly in the platform, or a reference to an external URL — all three types are stored, organized, and searched in exactly the same way. Creating a record manually (rather than via file upload) is useful when you want to capture a written summary, log a decision, or bookmark an important web resource alongside your other archived documents.

## Creating a Record

<Steps>
  <Step title="Open the New Record dialog">
    Click the **+ New Record** button, which appears in both the top navigation bar and the left sidebar. The dialog opens in the center of the screen.
  </Step>
  <Step title="Choose the record type">
    Select one of the three available record types:
    - **File Upload** — attach a file from your device (see [Uploading Files](/records/uploading-files) for full details)
    - **Text Note** — compose a formatted note directly in ORArchive
    - **External Link** — save and preview a URL as a record

    Your selection determines which fields and editor appear in the next step.
  </Step>
  <Step title="Fill in the record details">
    Complete the fields in the record form:
    - **Title** *(required)* — a clear, descriptive name for the record
    - **Description** — a summary of the record's content or purpose
    - **Category** — choose from your workspace's existing categories or create a new one
    - **Tags** — add one or more tags to make the record easier to filter and retrieve

    The Title field is the only required field; all others are optional but strongly recommended for searchability.
  </Step>
  <Step title="Save the record">
    Click **Save Record**. ORArchive creates the record immediately and opens its detail panel, where you can review and further edit any field at any time.
  </Step>
</Steps>

## Record Types in Detail

### Text Notes

When you select **Text Note**, a full rich-text editor appears below the metadata fields. You can write freeform content and apply formatting such as bold, italic, headings, bullet lists, numbered lists, and inline code. Paste content from other applications and the editor preserves basic formatting automatically.

Text notes are **fully indexed and searchable** — every word in the body of your note is included in ORArchive's search index, so you can find it later by searching for any phrase it contains. Notes can also be exported as PDF or plain text from the record detail panel.

### External Links

When you select **External Link**, a URL input field appears. Paste the full URL of the page you want to archive, then click **Fetch Preview**. ORArchive retrieves the page's title, description, and thumbnail image automatically and populates the record fields with that information. You can override any of these values before saving — for example, replacing an auto-generated title with a more descriptive one that fits your archive's naming conventions.

The original URL remains stored on the record and is displayed as a clickable link in the detail panel. ORArchive does not cache the remote page's content, so the link reflects whatever is currently live at that address.

<Tip>
  Adopt a consistent naming convention for record titles from the start — for example, prefixing titles with a date (`2025-06 Board Meeting Notes`) or a project code. Consistent patterns make filtering, sorting, and bulk operations significantly faster as your archive grows.
</Tip>

<Note>
  Every record creation event is logged automatically in the **Activity Feed** (accessible from the left sidebar). The log entry includes the record title, type, creator, and timestamp, providing a full audit trail for your workspace.
</Note>
