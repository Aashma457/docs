ORArchive makes it straightforward to keep your archived records accurate and up to date. You can edit a record's metadata at any time without affecting its file content, replace a file with a newer version while retaining the full history, or revise the body of a text note — all from the same record detail panel. Changes are saved individually for each field so you never risk losing work across unrelated parts of a record.

## Editing Metadata

Metadata includes a record's title, description, category, and tags — the fields that determine how the record is organized and discovered within your archive.

<Steps>
  <Step title="Open the record detail panel">
    Click on any record in the main content area. The detail panel slides in from the right side of the screen, showing all current metadata and a preview of the record's content.
  </Step>
  <Step title="Enter edit mode">
    Click the **Edit** (pencil) icon in the top-right corner of the detail panel header. All editable metadata fields become active input controls.
  </Step>
  <Step title="Update the fields">
    Make changes to any combination of the following fields:
    - **Title** — rename the record to reflect updated content or improved naming conventions
    - **Description** — revise the summary to keep it accurate
    - **Category** — reassign the record to a different category if its classification has changed
    - **Tags** — add new tags or remove existing ones by clicking the × on any tag chip
  </Step>
  <Step title="Save your changes">
    Click **Save Changes**. The detail panel returns to view mode and displays your updated values immediately. The edit is also logged in the record's activity history.
  </Step>
</Steps>

## Replacing a File

If a document has been updated — for example, a contract has been revised or a report has been finalized — you can replace the file attached to an existing record without losing the record's metadata or history.

While in edit mode on the record detail panel, click **Replace File**. The file picker opens; select the new version of the file from your device and confirm. ORArchive uploads the replacement file and associates it with the same record, keeping your title, description, category, and tags intact.

<Note>
  Replacing a file does **not** delete the previous version. Every version of the file is preserved and accessible from the **History** tab of the record detail panel. Each entry in the history shows the file name, upload date, and the user who uploaded it. You can restore any earlier version at any time by opening it in the History tab and clicking **Restore This Version** — the restored file becomes the current version, and the one it replaced is added back into the history log.
</Note>

## Editing a Text Note

For records of the **Text Note** type, you can revise the body content at any time independently of the record's metadata.

Open the record detail panel and click **Edit Content**. The rich-text editor opens with the current note content loaded. Make your changes — add, remove, or reformat any text — then click **Save** to commit the updated version. As with file replacements, the previous version of the note is preserved in the History tab, allowing you to review or restore earlier drafts if needed.

<Warning>
  Changing a record's **Category** may cause it to disappear from filtered views that other users in your workspace have saved. For example, if a colleague has a saved search pinned to the sidebar that filters by "Legal Documents" and you move a record out of that category, it will no longer appear in their view. Communicate category changes to affected team members before saving them.
</Warning>
